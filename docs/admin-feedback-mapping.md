# Admin Feedback Mapping

来源文档：

- `/Users/ruanhaokang/Downloads/DA 网站讨论 3.27.docx`

本文件只整理 `admin` 端，不处理客户端。

## 设计目标

新版 `admin-web-next` 不再按“后台页面集合”实现，而是按“招聘运营工作台”实现。

核心流程：

1. 进入岗位
2. 查看该岗位招聘进展
3. 在阶段表格中直接处理人选
4. 候选人详情作为辅助视图
5. 设置页用于规则配置，而不是高频操作

## 一级模块

建议的一级导航：

1. 工作台
2. 岗位管理
3. 招聘进展
4. 总人才库
5. 邮件与模板
6. 系统设置

## 文档需求映射

### 1. 工作台

文档反馈：

- 需要按周 / 月查看核心数据
- 指标包括：测试题总量、申请、成功签约、在职等

新项目实现建议：

- 顶部指标卡
- 趋势图按周 / 月切换
- 最新动态流

### 2. 岗位管理

文档反馈：

- 岗位详情缺少“申请情况总结”
- 列表页筛选能力不足，应支持公司、在招状态、岗位名称、创建时间
- 创建岗位页需要 C 端预览
- 岗位描述希望补充更丰富编辑能力
- 复制岗位与报名表单、自动筛选条件的继承关系需要明确
- 第二步需要勾选筛选条件
- 发布岗位后应跳转到岗位详情页

新项目实现建议：

- 列表页：搜索 + 多条件筛选 + 排序 + 列配置
- 详情页：总览 + 申请情况总结 + 配置摘要 + 招聘进展入口
- 创建页：按步骤重构为基础信息 / 表单与筛选条件 / 预览与发布

### 3. 总人才库

文档反馈：

- 需要作为“不分岗位的总人才池”
- 缺少排序、筛选、搜索、分页导航

新项目实现建议：

- 统一候选人资产列表页
- 支持多字段搜索、筛选、排序、分页、批量操作

### 4. 招聘进展

文档反馈：

- 需要“查看岗位详情”功能
- 各阶段都需要筛选、排序、搜索、分页导航
- 从阶段点进候选人详情后，返回应回到原阶段，而不是总人才库
- 希望更接近展开式操作，不希望总是跳到新页面
- 测试题回收、筛选通过、合同库、在职、淘汰各阶段都需要默认字段和批量操作
- 附件类字段希望直接在表格里预览/上传，而不是藏进操作菜单
- 需要“所有申请人”列表
- “招聘流水线”统一改名为“招聘进展”

新项目实现建议：

- 招聘进展作为一级模块
- 页面内再选择岗位
- 结构：岗位摘要 + 阶段 tabs + 高密度操作表格
- 支持上下文返回
- 支持列内操作、附件预览、批量操作、冻结列、列宽拖拽

### 5. 测试题回收阶段

文档反馈：

- 测试结果应直接在表格中完成
- 测试结果选项：通过 / 待定 / 不通过 / 需重新提交
- 测试评价应为自由输入文本
- “查看测试题”要直接成为列，不放到操作里
- 增加默认字段“判题负责人”
- 支持批量设置“判题负责人”

新项目实现建议：

- 测试题回收阶段表格支持列内编辑
- 测试题附件直接预览
- 结果、评价、负责人支持单行与批量修改

### 6. 筛选通过与淘汰

文档反馈：

- 增加字段：质检、质检反馈
- 当质检为“待返修”时，需要通知判题负责人
- 支持批量移回测试题回收
- 接受时薪为内部数字字段，美元单位
- 接受时薪希望支持条件格式
- 增加“签约进展”字段与批量修改
- 待签合同和测试题附件应直接显示在表格字段中

新项目实现建议：

- 在阶段表格中引入列配置与条件格式
- 支持状态字段、质检字段、签约进展字段列内编辑
- 支持批量流转回测试题回收

### 7. 合同库

文档反馈：

- 增加默认字段：合同审核
- 待签合同 / 签回合同 / ID 附件直接预览
- “标记在职”改为“成功签约”
- 点击成功签约自动进入在职
- 支持批量成功签约

新项目实现建议：

- 合同库作为阶段表格继续承载列内附件操作
- 成功签约作为显式流转动作

### 8. 在职

文档反馈：

- 增加“入职进展”
- 支持批量修改入职进展

新项目实现建议：

- 在职阶段新增状态字段和批量操作区

### 9. 候选人详情

文档反馈：

- 基本信息中缺少简历附件，且应支持下载 / 预览
- 最低期望时薪应跟岗位，而不是跟人名
- 需要从招聘进展状态定位到该岗位下的人选列表
- 需要看到与某岗位相关的阶段信息与合同/补全信息
- 操作记录不应是一条无限增长总流水，应按岗位查看

新项目实现建议：

- 候选人详情重构为“基本信息 + 岗位关联 + 阶段信息 + 附件 + 分岗位操作记录”

### 10. 邮件与模板

文档反馈：

- 附件应支持多文件上传
- 支持变量替换与占位配置
- 模板与签名需要拖拽排序

新项目实现建议：

- 邮件与模板作为独立一级模块
- 模板树 + 编辑器 + 变量配置 + 多附件

### 11. 系统设置与全局能力

文档反馈：

- 所有表格支持拖拽列宽
- 表格支持冻结首行首列
- 自动触发流程、自动邮件设置
- 功能权限

新项目实现建议：

- 从底层表格组件开始统一支持列宽调整、冻结列、列配置
- 自动化与权限配置放入系统设置

## 分阶段实施建议

### Phase 1

- 新版 Layout 与导航
- 工作台
- 岗位管理列表与详情
- 招聘进展基础框架
- 总人才库基础框架

### Phase 2

- 招聘进展：搜索、筛选、排序、分页、上下文返回
- 测试题回收的列内编辑
- 所有申请人页签

### Phase 3

- 筛选通过 / 淘汰 / 合同库 / 在职的关键业务字段
- 批量操作
- 附件列内预览与上传

### Phase 4

- 候选人详情重构
- 岗位创建页重构
- 邮件模板、签名、表单策略
- 自动化与权限
